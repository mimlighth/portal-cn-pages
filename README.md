# portal-cn-pages

本仓库用于归档和发布多个独立 HTML 页面，不针对任何单个域名或具体网站。仓库中的页面以静态 HTML 文件的形式存在，可直接通过浏览器访问，也可用于快速部署至各类静态托管平台。

## 项目简介

portal-cn-pages 是一个通用型 HTML 页面集合仓库，主要用于存放和分发各类独立页面。这些页面可能包含导航、信息聚合、工具界面或其他功能型或展示型界面。所有页面以纯前端静态文件形式存在，不依赖后端服务或特定运行环境。

## 目录说明

```
portal-cn-pages/
├── pages/           # 存放所有独立 HTML 页面
├── assets/          # 页面引用的公共资源（CSS、JS、图片等）
├── docs/            # 额外文档或说明文件（可选）
└── README.md        # 本文件
```

- `pages/`：每个子目录或文件对应一个独立页面，文件名即为页面标识。
- `assets/`：存放多个页面可能共享的样式、脚本及图片资源，便于统一管理和维护。
- `docs/`：可存放与页面使用或维护相关的补充文档。

## 页面归档说明

- 所有页面均为标准 HTML5 格式，推荐使用相对路径引用资源。
- 页面之间相互独立，不共享状态或数据。
- 每个页面应保持自包含，如需外部资源，请放置于 `assets/` 目录下并正确引用。
- 归档时建议在页面文件名或注释中注明版本或更新日期，便于追踪。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件，并根据需要更新 `assets/` 中的资源。
- 更新页面：直接修改对应 HTML 文件，建议保留修改记录（如使用 Git 提交信息）。
- 删除页面：移出 `pages/` 中对应文件，并检查是否需清理相关资源引用。
- 如对仓库结构或页面组织有改进建议，欢迎提交 Issue 或 Pull Request。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件查看效果。
3. 也可将整个仓库部署至任意静态文件服务器或托管平台，通过 URL 访问页面。
